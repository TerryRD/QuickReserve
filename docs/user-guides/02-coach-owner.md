# 🏓 教練 Owner 使用手冊

你是租戶的擁有者。你可以管理服務、行事曆、預約、助教。

## 登入

平台管理員會給你一個邀請連結（長得像 `quick-reserve-mu.vercel.app/invite/abc...`）。

1. 開那個連結
2. 若你還沒帳號 → 註冊（系統會自動把你帶回 invite 頁）
3. 若已有帳號 → 登入
4. 按「接受邀請」 → 進入你的後台 `/dashboard`

> **測試帳號**：`demo-coach-wang@example.com` / `Test1234!`

---

## 1. 儀表板（/dashboard）

進來看到：
- **公開連結**（你的 `/your-slug`）← 把這個傳給學員
- **2 個統計**：待確認預約數、本週時段數
- **2 個捷徑卡**：行事曆、服務管理
- **即將到來的預約** ← 最近 5 個

## 2. 服務項目（/services）

學員預約時必須先選一個「服務」。

### 新增服務

右上「+ 新增服務」 → 填：
- 名稱（必填，例如「桌球 1 對 1 課」）
- 描述（選填，告訴學員這堂課做什麼）
- 時長（分鐘，必填）
- 價格（可空 = 「洽詢」）

### 編輯 / 停用

每張服務卡片右下「編輯」打開原本表單修改。
停用後不會出現在公開頁，但既有預約保留。

### 拖曳排序公開頁顯示順序(A-9, 2026-05-29)

在「ALL」tab(未篩選)且有 2 個以上服務時,每張卡片右上角會出現灰色拖曳把手(⋮⋮)。

- 拖把手 → 換位置 → 鬆開即儲存(會 toast 失敗訊息,沒 toast = 成功)
- 公開頁 `/<你的 slug>` 的服務區塊會依此順序顯示
- 1-on-1 / Group tab 或刪除檢視不啟用 drag(避免跨類別亂排)
- 拖曳啟用門檻是 6px 移動,所以點「編輯/封存」按鈕不會誤觸 drag

> Staff 助教只能看不能改 — 服務由 Owner 統一管理。

## 3. 行事曆（/calendar）

### 切換視圖（FR-102）

行事曆頂端有「週 / 日 / 列表」三顆 tab：
- **週**（預設）：完整 7 日網格，最適合排班全局檢視
- **日**：聚焦單日，URL 帶 `?date=2026-05-21`；上下箭頭以「天」為單位翻頁
- **列表**：依日期分組的線性清單，適合手機 / 平板快速掃描

### 看自己的時段

預設顯示本週，週一到週日。顏色：
- 🟦 藍 = 可預約
- 🟧 橘 = 待確認（學員預約後但你還沒按確認）
- 🟩 綠 = 已確認
- ⬜ 灰 = 已取消

> **團班的 slot 狀態(2026-05-29 修正)**:max_capacity > 1 的服務 slot,只在**滿了**才會變橘色/綠色;報名人數還沒到上限時持續維持藍色「可預約」,讓後面的學員還能加入。確認狀態仍依 min_attendance:到了下限會自動 confirm 所有 pending bookings,但 slot 仍可繼續加人到 max。詳細狀態機在 `README.md`「Slot lifecycle」。

點側邊的左右箭頭切換週次，「回本週」回今天。

### 新增單一時段

右上「+ 新增時段」 → 在 dialog 中選：
- 服務（從你建好的服務清單）
- 日期
- 開始 / 結束時間

按「新增」。
**衝突偵測**：如果跟既有時段重疊，會顯示「與以下時段衝突」並列出衝突的時段。

### 批量 + 重複規則（⚡ 批量/重複）

右上「⚡ 批量 / 重複」按鈕 → 彈出更大的 dialog：

1. **選服務**
2. **重複方式**：每天 / 每週 / 每月 / 每 N 天
3. **若選每週**：勾選星期幾（一三五、二四六等）
4. **若選每月**：選每月幾號
5. **開始日期 + 起迄時間**
6. **何時結束**：
   - `N 次後`（例如連續 12 週）
   - `截止日` 之前
   - `不限` ← 系統每天自動往前推 1 天，永遠維持 90 天可預約

**即時預估(A-3, 2026-05-29):** 底部 PREVIEW 區塊邊填邊算,300ms debounce 後顯示「共 N 個時段 · 其中 K 個衝突 · M 個落在作息外」。讓你在按「建立」前就知道規則會撞到什麼,可以即時調整 start_time/end_time 或勾選星期幾。

按「建立」 → 若沒衝突直接建好;若有衝突會在 dialog 內列出衝突清單,你可以:
- 點每筆衝突右邊「跳轉去調整」→ 直接跳到那一週的行事曆
- 或點「略過衝突日,建立其他」 → 自動跳過衝突日、其他都建

### 多成員視圖（Owner 限定，FR-077）

如果你有助教，行事曆上方會出現「檢視」按鈕，點開有 checkbox 下拉清單，可以：
- **預設**：看自己 + 全部助教（slot 上會顯示成員名稱 badge）
- **取消勾選**特定助教 → 只看選的人
- **單獨看自己**：取消所有助教 → UI 變回「我的視圖」可以新增時段

其他成員的時段在你的視圖中為**虛線邊框 + 唯讀**，點開只能看不能改。

### 點擊行事曆上的時段（FR-080）

每個 slot 都可點擊 → 跳出 detail 視窗：
- 顯示：服務、時間、負責成員、狀態、學員（若已被預約）
- 若是**自己的可預約時段** → 顯示「刪除時段」按鈕
- 若是 `pending` / `booked` 狀態 → 顯示「前往預約管理」連結

### 管理重複規則（FR-073~076）

行事曆頁右上有「管理重複規則」按鈕 → `/calendar/rules`

每筆規則卡片顯示：
- 服務名稱 + 規則摘要（例「每週一三五 19:00–21:00 共 12 次」）
- 起始日期 + 結束條件
- **未來剩餘時段數**
- 右側：**啟用 / 停用** toggle + **刪除** 按鈕

**刪除規則時**會彈出對話框，可勾選：
- 「同時刪除未來尚未被預約的時段」（預設勾選）

⚠️ 已被學員預約（pending / booked）的時段**不會**被刪除，紀錄保留。

## 4. 預約管理（/bookings）

學員按下「預約」後，會出現在這裡，初始狀態 `待確認`（pending）。

### 篩選

頂部 5 顆按鈕：全部 / 待確認 / 已確認 / 已完成 / 已取消。

### 確認預約

每張「待確認」卡片右側有兩顆按鈕：
- **確認** → 預約變「已確認」、對應時段變綠
- **取消** → 預約變「已取消」、時段釋出回「可預約」

學員那邊會收到對應的 Web Push 通知（如果他啟用過）。

### 取消已確認預約

「已確認」狀態也能取消（教練有事不能上課）。一樣按「取消」即可。

## 5. 助教管理（/staff，僅 Owner）

### 邀請助教

填 email → 送出 → 拿到 invite URL → 傳給助教。

助教接受後會出現在下方列表，狀態 `已加入`。

### 移除助教

每位 staff 右側「移除」按鈕。這只是把 status 改成 `removed`（軟刪除），歷史資料保留。

## 6. 通知（兩個獨立路徑）

### 6.1 通知收件匣（/notifications）

新版的 `/notifications` 是**通知記錄列表**(notification_log inbox),不是設定頁。會列出所有寄給你的事件:新預約、改期、套裝申請、週報等。

- 上方 tab 切換:全部 / 預約 / 套裝 / 系統
- 未讀通知左側顯示**黃色側條 + 字重粗**(由 DB `read_at IS NULL` 決定,A-5 改為持久化)
- **點任一未讀通知** → 自動標為已讀(optimistic UI 立即淡出,後端 server action 同步)
- 標題右側「全部標為已讀(N)」按鈕一鍵清空所有未讀(N 是未讀數;為 0 時按鈕隱藏)
- 右上有「推播偏好」按鈕跳到設定頁(下一節)

### 6.2 通知偏好（/settings/notifications）

#### 啟用 Web Push

進這頁會看到「此裝置訂閱」卡 → 點「啟用通知」 → 瀏覽器彈出權限詢問 → 同意。

之後新預約申請、學員取消等事件會即時推播。下方「所有裝置」列出你目前已訂閱的瀏覽器/裝置,可以個別移除。

#### 事件 × 通道矩陣

9 個事件 × 2 個通道的勾選矩陣,每格獨立可開關:

| 事件 | Web Push 推播 | In-app 站內 |
|---|---|---|
| 新預約 / 預約已確認 / 預約取消 / 預約改期 |  |  |
| 套裝申請 / 套裝核准 |  |  |
| 課前提醒 / 每日提醒 / 每週摘要 |  |  |

> Email 通道**本期不做**(成本/流量限制決策,列 Phase 2 backlog)。

#### 勿擾時段

可選的時段(例: 22:00 → 07:00),這段時間內的推播會延後到結束時段後發送。跨午夜會正確處理。底下 checkbox 取消可整個停用勿擾。

按「儲存設定」一次寫入 `notification_preferences.channels` jsonb + `quiet_hours_start/end`。

## 6.5 服務啟用 / 停用 (FR-097)

編輯既有服務時，最下方新增「啟用此服務」勾選框。停用後：
- 公開預約頁不顯示
- 既有預約保留

## 7. 學員（/customers，FR-070~072）

側邊「學員」進入，看到所有曾跟你的租戶預約過的客戶。

每張卡片顯示：
- 姓名 + 大頭貼
- **總預約數 / 待確認數 / 已完成數**
- 最近預約日期

> 不同教練的學員資料**完全隔離**（透過 `tenant_customers` bridge + RLS 強制）。

要看單一學員的所有預約細節，請到「預約管理」用 filter 篩選。

### 封鎖學員 (FR-088~091)

每張學員卡片右下「封鎖」按鈕。封鎖後：
- 該學員無法在你的公開頁建立**新**預約（書系統會拋 `CUSTOMER_BLOCKED`）
- **既有**預約不受影響
- 已封鎖的學員卡片變紅色，可以「解除封鎖」

## 7.5 租戶資料 (FR-100, /settings/profile)

側邊「租戶資料」（僅 Owner 看得到）→ 編輯：
- **租戶名稱**（公開頁 hero 大字）
- **描述**（公開頁 hero 副標，建議 1~2 句你想讓學員第一眼看到的話）
- **聯絡方式**：Email / 電話 / LINE ID / 自由備註

填好按「儲存」即時生效，公開頁 `/your-slug` 會在 hero 區顯示已填寫的聯絡方式（未填的欄位自動隱藏，Email/電話可直接點擊呼叫）。

## 7.6 套裝管理 + 審核佇列(S4 + 2026-05-29 更新)

側邊「套裝」(`/packages`)管理你提供的套裝方案;「待審核」(`/packages/pending`)處理學員的購買申請。

### 套裝管理(/packages)

頂部 tab:**ALL / 1-ON-1 / GROUP / DRAFT**。Card 顯示:
- 套裝名 + 堂數 + 價格 + 期限天數
- 🟡 **POPULAR** pill(若被你勾選),公開頁會用同色 highlight
- 草稿 badge(尚未發布)

右上「+ 新增套裝」開 dialog。可指定:
- 對應服務、堂數、價格、有效期天數
- **POPULAR** 勾選(突顯精選)

### 套裝審核(/packages/pending)

學員按「申請此套裝」後出現在這裡,初始 `pending_review`。

4 個 KpiCard:總待審 / 等待最久 / 本週新進 / 本月通過。第一筆(最舊)會有黃色 ring 強調提醒。

每張審核卡顯示:
- 學員姓名 + **付款狀態 pill**(三種顏色)
  - 🟢 **已付款**(claimed_paid)— 綠
  - 🔵 **部分付款**(partial_paid)— 藍(A-4, 2026-05-29 加)
  - 🟡 **未付款**(awaiting_payment)— 黃
- 想買的套裝 + 堂數
- **RECEIPT block**(灰底 inline 框,A-4 加)— 若學員有填付款備註(例如「轉帳末四碼 1234」或「已付 5000 / 剩 5000 預約後付」),會顯示在這裡讓你比對銀行紀錄
- 申請時間

兩顆按鈕:
- **確認**:approve,套裝立即生效,學員可預約。`classes_used = 0`, `expires_at = now() + expires_in_days * 24h`
- **拒絕**:dialog 要你填原因(必填),學員收到通知含原因。Status → `rejected`,可留歷史紀錄

## 8. 公開連結

頂部 sidebar 底部有「🔗 公開預約頁 /your-slug」 → 點開複製 URL 給學員。

學員拿到後：
1. 開連結 → 看到你的介紹頁
2. 選服務 → 選日期 → 選時段 → 必須登入 / 註冊 → 送出預約
