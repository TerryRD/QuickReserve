# 👤 學員 / 客戶 使用手冊

你是要預約教練的人。

## 註冊 / 登入

### 從教練的公開頁進

直接點教練給你的 `/<slug>` 連結 → 找到時段按「前往預約」 → 系統會自動帶你去 `/login`,登入後回到原本的時段。

### 註冊新帳號(/signup)

填姓名、Email、密碼。

> **密碼強度檢查(2026-05-29 加)**:系統會擋掉常見字典詞、純數字、鍵盤序列等弱密碼,如果跳「密碼太弱,請避免常見字典詞、純數字或鍵盤序列」就請改強一點(例如混合大小寫 + 數字 + 符號,或一句容易記的英文短句)。

### 教練邀請(invite)

教練如果直接邀請你加入,連結會帶 `?invite=<token>` query。register 頁頂部會出現黃色 banner「**INVITED · 教練名稱**」(A-7, 2026-05-29 加,以前只顯示「INVITED」沒有名字),完成註冊後自動建立你跟該教練的關係,可立即購買他的套裝。

## 第一次預約

1. 教練給你一個專屬連結，例如：
   - `quick-reserve-mu.vercel.app/demo-wang-coach`
2. 開啟連結 → 看到教練介紹頁（hero 區會看到教練的描述與聯絡方式 — Email / 電話 / LINE，可直接點擊）

頁面上會有三步：

### Step 1 — 選擇服務

教練提供的所有服務（例如「桌球 1 對 1 課 60 分 $1500」），點一個。

> 服務顯示順序由教練自訂(A-9, 2026-05-29)— 不再依字母排,而是教練在後台拖曳卡片決定的優先順序。

### Step 2 — 選擇日期

橫向 **DateRibbon**(可滑動)。每個日期 chip 顯示星期 + 日期數字 + 可用時段數;沒時段的日期會 disable(灰底斜線)。點要的日期。

想看更遠的日期?頭尾「◄」「►」往前後翻週,中間「今天」直接跳回本日。

### Step 3 — 選擇時段

當日所有時段以 **TimeChip** pill 顯示,4 種 state:
- **open**(可預約)— 一般 outline,點擊選取
- **selected**(已選)— 黑底白字
- **group N/M**(團班,A-6 2026-05-29 開始真正連線)— 右側黃色 badge 顯示**已報/上限**;團班 slot 即使已有人報名,只要還沒滿就還能加入
- **full**(滿額)— 由 API 過濾掉不會顯示

選好時段後下方會出現**黑底 selected slot recap bar** 顯示「日期 時間 · 服務 · 時長」+ 大顆「前往預約」按鈕(改期模式則顯示「改期到此時段」)。點下去進「確認預約」頁。

### 確認預約頁

顯示：日期、服務、時長、價格。

- 若你**未登入** → 系統跳轉到 `/login`，登入或註冊後自動回來
- 若你**已登入** → 進入「預約確認」頁

「PACKAGE · 套裝餘額」區塊(A-1, 2026-05-29 更新)— 你的有效套裝會以 radio cards 列出,每張卡顯示套裝名 / 剩餘堂數 / 期限 / 進度條:
- **預設挑最快到期的**(避免某張套裝過期沒用)
- 但你可以**手動切換**到任何一張有效套裝(以前是 disabled)
- 沒有任何有效套裝時會跳 EmptyState「尚無可用套裝」+ 連結「瀏覽套裝」

填備註(選填)→ 按「送出預約申請」。

送出後狀態為**待確認**,教練確認後變「已確認」。
若中間套裝突然失效(例如剛好用完最後一堂被別的預約搶走),會 toast「選擇的套裝不可用,請重新選擇」讓你重選。

## 我的預約（/my-bookings）

登入後左上頭像可進去。

頁面上方有「**我的教練**」聚合區（FR-109）— 按教練分組顯示每位教練的累計預約次數，以及還沒完成的預約數，點擊可直接跳回該教練公開頁再次預約。

下方會列出所有預約：

| 狀態 | 顏色 | 意義 |
|------|------|------|
| 待確認 | 🟧 黃 | 教練還沒按確認 |
| 已確認 | 🟩 綠 | 教練同意，正式成立 |
| 已完成 | ⬜ 灰 | 課程已上完 |
| 已取消 | ⬜ 灰刪除線 | 你或教練取消了 |

每張卡片若「課程時間還沒到」會有兩顆按鈕：
- **改時間**（FR-092~096）：跳轉到該教練公開頁，進入「改期模式」 → 選新時段 → 系統會原子性取消舊預約並建立新預約（同一個 transaction）。不可跨教練改期。
- **取消預約**：直接釋出時段。

## 啟用 Web Push 通知

進 `/my-bookings` 上方會看到「啟用瀏覽器推播通知」橫幅。

- 點「啟用通知」 → 瀏覽器跳出權限請求 → 同意
- 之後預約被確認 / 取消 / 改期會即時通知

## 申請套裝(/<slug>/packages)

打開教練的 `/<slug>/packages` 看到所有方案(熱門方案會掛 🟡 **POPULAR** pill)。展開要的套裝 → 「申請此套裝」。

填:
- **付款狀態**(三選一,A-4, 2026-05-29 更新):
  - **已轉帳 / 已付款**(claimed_paid)
  - **已部分付款**(partial_paid)— 預付一部分,剩下後付
  - **未付款**(預約後再付)
- **付款備註**(選填,只在前兩種狀態顯示):寫銀行帳號末四碼、或「已付 5000 / 剩 5000 預約後付」之類,教練核對時看得到

「送出申請」→ 進 pending 狀態,教練那邊會收到通知。

## 我的通知設定

**通知設定**(`/account/notifications`):點 my-bookings 頁右上「⚙️ 通知」可以選:
- 每週日 20:00 收到下週預覽
- 每天早晨提醒今日行程
- 預約前 N 分鐘提醒（30 分 / 1 小時 / 1 天前等）
- 預約狀態變更立即通知

> 路徑備註:學員的通知偏好在 `/account/notifications`(獨立於 `/settings/notifications`,後者是教練後台用)。

## 試試看（用測試資料）

> 測試帳號：`demo-student-ming@example.com` / `Test1234!`

1. 開 `/login` → 用上面帳號登入
2. 自動進 `/my-bookings`：你會看到 2 筆預約
   - 王教練（待確認）
   - 林教練（已確認）
3. 開新分頁 `quick-reserve-mu.vercel.app/demo-chen-coach` → 選「高爾夫初學」→ 選一個有時段的日期 → 選時段 → 送出預約
4. 回 `/my-bookings` 應該看到新的一筆「待確認」
5. 點該預約的「取消預約」 → 即時消失（變已取消）

### 試試衝突偵測

開兩個 incognito 視窗：
1. 視窗 A：用 `demo-student-hua@example.com` 登入
2. 視窗 B：用 `demo-student-mei@example.com` 登入
3. 兩個視窗都打開 `/demo-wang-coach` → 都選**同一個**可用時段
4. 視窗 A 先按「送出預約」 → 成功
5. 視窗 B 也按「送出預約」 → 應該看到「該時段已不可預約」錯誤

這證明系統的原子鎖定有效，不會超賣。
