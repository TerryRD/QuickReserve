# 🛠️ 平台管理員 使用手冊

你是整個 SaaS 平台的擁有者。你的後台位於 `/platform/dashboard`。

## 登入

| 欄位 | 值 |
|------|----|
| 網址 | https://quick-reserve-mu.vercel.app/login |
| Email | `terry@gmail.com` |
| 密碼 | （你自己設的） |

登入後系統會自動把你導向 `/platform/dashboard`。

---

## 1. 儀表板（首頁）

側邊選單第一項。看到 6 個統計卡：
- 啟用租戶（active / 全部）
- 使用者總數
- 預約紀錄總數
- 待確認預約
- 推播訂閱數
- 平台管理員數

**用途**：日常監測。如果待確認預約堆積很久，可能有教練忘了確認。

---

## 2. 租戶管理（/platform/tenants）

### 邀請新教練

最上方那張「邀請新教練」卡片：

| 欄位 | 範例 |
|------|------|
| 教練 Email | `coach@example.com` |
| 租戶名稱 | `王教練` |
| Slug（公開連結） | `wang-coach`（只允許小寫英數 + 短橫線） |

按「送出邀請」→ 系統會顯示一個 **invite URL**，把這個 URL 傳給教練。

⚠️ 邀請有效期 **7 天**。若過期需重新邀請（目前沒有 UI，重新邀請會建第二個 tenant — 後續可改善）。

### 暫停 / 啟用租戶

下方列表每筆右側有「暫停」或「啟用」按鈕。

**暫停的效果**：
- 公開預約頁 `/{slug}` 顯示「服務暫停中」
- 教練本人登入後看到「您的租戶已被暫停」
- 既有預約**不取消**（學員的歷史紀錄保留）
- 但無法建立新預約

按「啟用」即可恢復。

### 進租戶詳情頁（FR-081~083）

點任一租戶的 slug 或名稱 → 進入 `/platform/tenants/[id]` 唯讀詳情頁：

- **4 個統計卡**：成員數 / 服務數 / 未來時段數 / 預約數（含待確認）
- **成員列表**：含 Owner + Staff，狀態與加入日期
- **服務列表**：含時長、價格、啟用狀態
- **最近 10 筆預約**：學員、服務、時間、狀態
- **學員 chip**：最多 50 位該租戶的學員

點右上「開公開頁」可在新分頁開啟教練的 `/{slug}` 預約頁。

> ⓘ 這是純唯讀視圖，不能代教練操作任何資料（刻意的權限邊界）。

## 3. 試試看（用測試資料）

> 假設你已照 README 的 seed 腳本建好測試資料。

### 試試暫停一個教練看看

1. 進 `/platform/tenants`
2. 點「王教練」右邊的「暫停」按鈕 → 確認
3. 新分頁打開 `/demo-wang-coach` → 應該看到「服務暫停中」
4. 回去把它「啟用」 → 再次刷新公開頁，恢復正常

### 試試邀請新教練

1. 在邀請表單填 email + name + slug（slug 要全新沒有用過的）
2. 拿到 invite URL
3. 開 incognito → 貼 invite URL → 進入註冊流程 → 完成註冊 → 接受邀請
4. 回到你的後台 `/platform/tenants` → 該租戶出現在列表

---

## 4. 重要原則

- **平台管理員不能直接代替教練操作**（不能代學員下訂、不能代教練確認預約） — 這是設計上刻意的權限邊界
- **要創更多平台管理員**：目前沒有 UI，要直接在 Supabase Dashboard 跑：
  ```sql
  insert into public.platform_admins (user_id) values ('<UUID>');
  ```
- **資料完全隔離**：你能「看到」所有租戶資料，但教練 A 看不到教練 B 的資料（PostgreSQL RLS 強制）

## FAQ

**Q：教練忘記登入密碼怎麼辦？**
A：目前無 UI 讓你直接重設。請教練自行用 `/login` 旁的「忘記密碼」（待實作） — 暫時方案：你在 Supabase Auth Dashboard 幫他重設。

**Q：我可以刪除一個租戶嗎？**
A：暫停即可；如果真的要刪除，需手動進 Supabase 跑 SQL `delete from tenants where id = ...`（會 CASCADE 刪除所有相關資料）。
